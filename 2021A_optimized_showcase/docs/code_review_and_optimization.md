# 原始代码详细阅读与优化映射

## 1. 阅读对象

- `第三问光线接受比代码.py`
- `图像.py`
- `main_code(2).m`
- `select_best_alteration(2).m`

## 2. 关键问题梳理

### 2.1 Python 脚本中的性能与鲁棒性问题

1. 大规模逐点 `for` 循环计算，采样间隔达到 `1e-4` 时点数极大，耗时高。
2. 固定硬编码路径（如临时目录、桌面路径），迁移到新机器后容易失效。
3. 同文件中重复读取与重复绘图逻辑，维护成本高。
4. 关键公式近零分母场景未做保护，存在数值不稳定风险。

### 2.2 MATLAB 主流程中的工程化问题

1. 脚本耦合度高，输入输出与计算混在一起。
2. 数据写回 Excel 逻辑与算法主流程紧耦合，不利于测试。
3. 部分计算步骤缺少中间量持久化，不便于复现实验轨迹。

### 2.3 可视化展示问题

1. 原图以临时分析为主，不是 GitHub 友好产物。
2. 缺少统一视觉风格与“封面级”动画。
3. 缺少自动化产图脚本，无法一键复现。

## 3. 对应优化策略

### 3.1 计算内核优化

- 将核心反射落点计算改为 NumPy 向量化。
- 保留循环版实现，用于性能基准与正确性对照。
- 对近零分母情况做掩码保护，避免不稳定点污染结果。

### 3.2 工程结构优化

- 新建 `src` 模块化目录，拆分为数据、模型、基准、可视化、管线五层。
- 新建 `outputs` 统一承接图像和 JSON 报告。
- 所有目录补齐 README，降低二次使用门槛。

### 3.3 数据读取优化

- 支持自动发现 `附件1.csv`。
- 支持多编码读取（UTF-8/GBK/GB18030）。
- 无数据时自动降级到合成节点云，保证流程可演示。

### 3.4 展示优化

- 生成 GitHub 封面 GIF：过程扫描 + 接收率动态对比。
- 生成参数扫描 GIF：展示 `h0` 对性能指标影响。
- 生成仪表板 PNG：一图覆盖主要结果。
